Cybersecurity

Security is as much about policy and process as it is about technology. In CYBR-111 I studied how organizations build security programs, write the policies that govern them, and align to established frameworks.

Framework

NIST 800-53 deep dive

Wrote an introduction to NIST Special Publication 800-53 (Revision 5) — the framework of security and privacy controls used across U.S. federal agencies and, increasingly, the private sector. It focuses on what outcomes to achieve rather than dictating how, organizing controls into 20 families like Access Control, Incident Response, and Configuration Management. Revision 5's big shift was merging security and privacy controls into one unified catalog.

Policy writing

Acceptable Use Policy — "Avengers Campus HQ"

Wrote a full Acceptable Use Policy, using a playful "Avengers Campus HQ" theme to cover the real structure of an AUP: purpose, scope, acceptable and prohibited use, and network/communication rules. (Yes — it prohibits x-ray vision in the locker rooms and using super-powers to bypass security controls.) A fun way to learn how real organizations define appropriate use of systems and facilities.

Process

Program closeout

Also worked through a security program closeout — documenting how a security effort is formally wrapped up, lessons captured, and controls handed off.
